## 结构化知识摘要（由 PDF 精确提取）

### 1. 流程范围

- `1.2a` 制定订单查货计划（正常订单，除联营业务/联营款订单）
- `1.2b` 制定订单查货计划（联营业务/联营款订单）

### 2. 输入与输出

- 输入资料
- 采购部订咮信息
- 工厂大货进度表
- 认可查货人员名单
- 认可生产工厂名单
- 免检及非免检工厂名单
- 查货人员邮箱地址
- 工厂邮箱地址
- 联营业务/联营款查货申请信息

- 输出结果
- 查货计划表
- 联营业务/联营款查货计划
- 查货通知

### 3. 流程活动与规则

- `1.2.1a 抽取/整理订单的查货排期资料`
- 适用于正常订单。
- 根据采购部订差信息及工厂大货进度表，质量管理部门按订单归属口岸，在 `B/S 查货系统` 中抽取相应订单的查货排期资料。

- `1.2.1b 整理联营业务/联营款订单查货资料`
- 适用于联营业务/联营款订单。
- 根据采购部在 `Lotus Notes` 或办公平台联营业务/联营款资料库发出的查货申请信息，由质量管理部门按订单归属口岸以电子邮件转发相关查货信息。

- `1.2.2a 制定订单的查货计划表`
- 适用于正常订单。
- 所属口岸在 `B/S 查货系统` 中制作每张订单的查货计划表。
- 计划表应包括：工厂、半成品查货或成品查货、查货员、查货日期等基本信息。
- 应按订单查货日期轻重缓急，对未来一周或数周作出查货计划，可按周或按月安排，并防止漏排。

- `1.2.2b 制定联营业务/联营款查货计划表`
- 适用于联营业务/联营款订单。
- 由质量管理部门所属口岸主管根据采购组发出的查货申请制作对应口岸的联营业务/联营款订单查货计划表。
- 计划表应包括：联营业务/联营款、成品查货、查货员、查货日期等基本信息。

- `1.2.3a 发出查货通知`
- 适用于正常订单。
- 质量管理部门所属口岸依据订单查货计划表，在 `B/S 查货系统` 中编制订单查货通知，并及时通知口岸查货人员及工厂负责人。

- `1.2.3b 发出查货通知`
- 适用于联营业务/联营款订单。
- 质量管理部门资料组按所属口岸编制联营业务/联营款订单查货通知，并及时通知口岸查货人员及联营业务/联营款负责人。
- 为避免漏排漏查，口岸在收到查货通知邮件后，需回复总部资料组确认已收到通知。

### 4. 角色分工

- 采购部：提供订单信息、工厂进度表或联营查货申请。
- 质量管理部门所属口岸：抽取排期资料、编制查货计划、发出正常订单查货通知。
- 质量管理部门资料组：编制联营业务/联营款查货通知。
- 口岸 QA / 查货人员：接收通知并执行后续查货流程。
- 工厂负责人或联营业务/联营款负责人：接收查货通知并配合安排。

### 5. KPI 指标

- 按时完成率
- 计算公式：`按时完成率 = 按时发出查货通知的单数 / 计划查货的单数 x 100%`
